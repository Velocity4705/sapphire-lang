2. Save IR to File

3. Compile IR to Native Code
   ```bash

   clang program.o -o program
   ./program
   ```

EXAMPLE: COMPREHENSIVE TEST

Input (comprehensive_test.spp):
```sapphire
let a = 100
let b = 50
let sum = a + b
let x = 3.14
let mixed = a + x
let is_greater = a > b
let both_true = is_greater and is_equal
```

Generated IR:
```llvm
define i32 @main() {
entry:
  %a = alloca i64, align 8
  store i64 100, ptr %a, align 4
  %b = alloca i64, align 8
  store i64 50, ptr %b, align 4
  %a1 = load i64, ptr %a, align 4
  %b2 = load i64, ptr %b, align 4
  %addtmp = add i64 %a1, %b2
  %sum = alloca i64, align 8
  store i64 %addtmp, ptr %sum, align 4
  %x = alloca double, align 8
  store double 3.140000e+00, ptr %x, align 8
  %a3 = load i64, ptr %a, align 4
  %promotetmp = sitofp i64 %a3 to double
  %x4 = load double, ptr %x, align 8
  %addtmp5 = fadd double %promotetmp, %x4
  %mixed = alloca double, align 8
  store double %addtmp5, ptr %mixed, align 8
  %a6 = load i64, ptr %a, align 4
  %b7 = load i64, ptr %b, align 4
  %cmptmp = icmp sgt i64 %a6, %b7
  %is_greater = alloca i1, align 1
  store i1 %cmptmp, ptr %is_greater, align 1
  ; ... short-circuit evaluation with PHI nodes ...
  ret i32 0
}
```

================================================================================
KNOWN LIMITATIONS
================================================================================

1. Function Calls
   - Parser doesn't fully support function syntax
   - Causes segfaults when compiling programs with print()
   - Workaround: Use interpreter mode for programs with functions

2. Direct Executable Generation
   - LLVM 21 API changes require updates
   - Workaround: Use external tools (llc + clang)
   - This is a common workflow and achieves same performance

3. Optimization
   - Disabled by default to show unoptimized IR
   - When enabled, removes unused variables (correct behavior)
   - Can be re-enabled for production use

================================================================================
WORKAROUND: NATIVE COMPILATION
================================================================================

Until direct executable generation is fixed, use this workflow:

Step 1: Generate IR
```bash
./sapp compile program.spp > program.ll
```

Step 2: Compile to object file
```bash
llc program.ll -o program.o
```

Step 3: Link to executable
```bash
clang program.o -o program
```

Step 4: Run!
```bash
./program
```

This achieves the same native performance as direct compilation!

Alternative: One-liner
```bash
./sapp compile program.spp > program.ll && llc program.ll -o program.o && clang program.o -o program && ./program
```
