================================================================================
LESSONS LEARNED
================================================================================

1. Opaque Pointers Work Well
   - Avoids exposing LLVM headers in public interface
   - Prevents compilation issues with forward declarations
   - Makes code cleaner and easier to maintain

2. Conditional Compilation is Essential
   - Not everyone has LLVM installed
   - Graceful fallback keeps project accessible
   - Clear error messages guide users

3. Documentation First Pays Off
   - Having a clear plan makes implementation easier
   - Phase-by-phase breakdown prevents overwhelm
   - Success criteria keep us focused

4. Build System Matters
   - Automatic detection is user-friendly
   - Clear status messages build confidence
   - Incremental compilation speeds development

================================================================================
CHALLENGES OVERCOME
================================================================================

1. LLVM Header Complexity
   Problem: Including LLVM headers caused compilation errors
   Solution: Opaque pointer pattern with void* impl_

2. Forward Declaration Issues
   Problem: unique_ptr with incomplete types failed
   Solution: Use opaque pointers, defer LLVM includes to .cpp

3. Build System Integration
   Problem: Need to support both with/without LLVM
   Solution: Conditional compilation with HAVE_LLVM flag

4. Keeping It Simple
   Problem: LLVM is complex, easy to over-engineer
   Solution: Start with stub, add functionality incrementally
