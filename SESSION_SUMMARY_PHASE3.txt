================================================================================
KEY DECISIONS
================================================================================

1. Memory Poisoning Strategy
   - Decision: Don't poison in free list
   - Reason: Would corrupt FreeNode structures
   - Solution: Use FREED_BIT flag instead
   - Result: Works perfectly with allocator

2. Integration Approach
   - Decision: Call mark_freed() in Allocator::free()
   - Reason: Automatic safety checks
   - Result: Seamless integration

3. Error Message Design
   - Decision: Include context (addresses, indices, types)
   - Reason: Make debugging easy
   - Result: Clear, actionable error messages

4. SafePtr Design
   - Decision: RAII wrapper with automatic checks
   - Reason: Safe by default
   - Result: Easy to use, type-safe

================================================================================
CHALLENGES OVERCOME
================================================================================

1. Memory Poisoning vs Free Lists
   - Challenge: Poisoning corrupted free list
   - Solution: Don't poison in mark_freed()
   - Result: Use FREED_BIT flag instead

2. Test Integration
   - Challenge: Tests calling both mark_freed() and free()
   - Solution: Updated tests to avoid double-free
   - Result: All tests passing

3. Allocator Dependencies
   - Challenge: Allocator tests needed safety.cpp
   - Solution: Updated Makefile dependencies
   - Result: All tests build and run
