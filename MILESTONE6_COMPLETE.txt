================================================================================
IMPLEMENTATION DETAILS
================================================================================

Thread (runtime/concurrency/thread.h, thread.cpp):
- Lightweight thread abstraction over std::thread
- Template-based spawn for any callable
- Move semantics (non-copyable)
- Thread IDs and naming
- ~150 lines of code

Mutex (runtime/concurrency/mutex.h, mutex.cpp):
- Mutual exclusion lock
- RAII Guard for automatic unlock
- Try-lock support
- ~80 lines of code

RwLock (runtime/concurrency/rwlock.h, rwlock.cpp):
- Reader-writer lock (multiple readers, one writer)
- ReadGuard and WriteGuard for RAII
- Try-lock for both read and write
- ~120 lines of code

Channel<T> (runtime/concurrency/channel.h):
- Template-based thread-safe message passing
- Buffered and unbuffered modes
- Blocking and non-blocking operations
- Channel closing with proper cleanup
- Header-only template (~200 lines)

ThreadPool (runtime/concurrency/threadpool.h, threadpool.cpp):
- Fixed-size pool of worker threads
- Task queue with condition variables
- Future-based result retrieval
- Wait for all tasks completion
- ~150 lines of code

================================================================================
USAGE EXAMPLES
================================================================================

Thread Spawning:
```cpp
Thread t = Thread::spawn([]() {
    std::cout << "Hello from thread!" << std::endl;
});
t.join();
```

Mutex Protection:
```cpp
Mutex m;
int counter = 0;
{
    auto guard = m.lock_guard();
    counter++;
}  // Automatically unlocked
```

Channel Communication:
```cpp
Channel<int> ch;

Thread sender = Thread::spawn([&ch]() {
    ch.send(42);
});

int value = ch.receive();
sender.join();
```

Thread Pool:
```cpp
ThreadPool pool(4);

auto future = pool.submit([]() {
    return expensive_computation();
});

int result = future.get();
```
