================================================================================
WHY THIS MATTERS
================================================================================

Memory management is one of the hardest problems in programming:

Problems with Manual Memory Management (C/C++):
❌ Memory leaks
❌ Use-after-free bugs
❌ Double-free errors
❌ Dangling pointers
❌ Buffer overflows
❌ Hard to debug
❌ Time-consuming

Problems with Simple GC (Java/Python):
❌ Long pause times
❌ High memory overhead
❌ Unpredictable performance
❌ Not suitable for real-time

Sapphire's Approach:
✅ Automatic (like Java/Python)
✅ Fast (like Go)
✅ Safe (like Rust)
✅ Optional manual control (like C++)
✅ Best of all worlds!

Technical Approach:
```
Object Layout:
+------------------+
| Type ID (8 bytes)|  <- What type is this?
+------------------+
| Size (4 bytes)   |  <- How big is it?
+------------------+
| GC Flags (4 bytes)| <- Mark bit, etc.
+------------------+
| Object Data...   |  <- Actual data
```

Allocation Strategy:
- Small objects (< 256 bytes): Size classes
- Large objects (>= 256 bytes): Direct allocation
- Thread-local: Fast path
- Global: Slow path with locking

Performance Target:
- Small allocation: < 100ns
- Large allocation: < 1μs
- Thread-local: < 50ns

================================================================================
TECHNICAL DETAILS
================================================================================

Allocator Interface:
```cpp
class Allocator {
public:
    // Allocate object of given size and type
    void* allocate(size_t size, TypeID type);
    
    // Free object (for manual management)
    void free(void* ptr);
    
    // Get object metadata
    ObjectHeader* get_header(void* ptr);
    
    // Statistics
    size_t total_allocated();
    size_t total_freed();
    size_t current_usage();
};
```

Object Header:
```cpp
struct ObjectHeader {
    uint64_t type_id;      // Type information
    uint32_t size;         // Object size
    uint32_t gc_flags;     // GC mark bit, etc.
    
    // Flags
    bool is_marked() const;
    void set_marked(bool marked);
    bool is_large() const;
};
```

Size Classes:
```
8, 16, 32, 64, 128, 256 bytes
Each has its own free list
Fast allocation from appropriate class
```

Thread-Local Allocation:
```cpp
thread_local BumpAllocator local_allocator;

void* fast_allocate(size_t size) {
    if (size <= 256 && local_allocator.has_space(size)) {
        return local_allocator.allocate(size);
    }
    return global_allocator.allocate(size);
}
```

================================================================================
INTEGRATION WITH CODEGEN
================================================================================

Current Codegen (Stack Only):
```llvm
%x = alloca i64, align 8
store i64 10, ptr %x, align 4
```

New Codegen (Heap Allocation):
```llvm
; Allocate object
%obj = call ptr @sapphire_allocate(i64 24, i64 %type_id)

; Store data
%data_ptr = getelementptr ptr %obj, i64 16  ; Skip header
store i64 10, ptr %data_ptr, align 8
```

Allocator Function:
```cpp
extern "C" void* sapphire_allocate(size_t size, uint64_t type_id) {
    return allocator.allocate(size, type_id);
}
```
