================================================================================
COMPILATION PIPELINE
================================================================================

Source (.spp)
    ↓
Lexer (tokens)
    ↓
Parser (AST)
    ↓
Type Checker (typed AST)
    ↓
LLVM Code Generator (LLVM IR)
    ↓
LLVM Optimizer (optimized IR)
    ↓
LLVM Backend (object file)
    ↓
System Linker (executable)
    ↓
Native Binary

================================================================================
EXAMPLE: END-TO-END COMPILATION
================================================================================

Input (simple.spp):
```sapphire
let x = 10
let y = 20
let sum = x + y
```

Step 1: Compile to IR
```bash
./sapp compile simple.spp > simple.ll
```

Step 2: Compile to object
```bash
llc -filetype=obj simple.ll -o simple.o
```

Step 3: Link to executable
```bash
clang simple.o -o simple
```

Step 4: Run
```bash
./simple
# Exit code: 0
```

Result: Native executable running at C-like speed! ✨

================================================================================
KNOWN LIMITATIONS
================================================================================

1. Function Calls (Parser Issue)
   - Parser doesn't fully support function syntax
   - Causes segfaults when compiling with print()
   - Workaround: Use interpreter mode
   - Fix: Update parser for 'fn' keyword

2. Direct Executable Generation (LLVM API)
   - LLVM 21 API changes require updates
   - Workaround: Use external tools (llc + clang)
   - This achieves same performance

3. Standard Library
   - No native print() in compiled mode yet
   - No file I/O in compiled mode yet
   - Next phase: Implement stdlib core

SUPPORTED FEATURES
================================================================================

✅ Variables
   let x = 10
   let name = "Alice"

✅ Arithmetic
   let sum = x + y
   let product = x * y
   let mixed = 10 + 3.14  # Type promotion

✅ Comparisons
   let is_greater = x > y
   let is_equal = x == 10

✅ Logical Operations
   let both = x and y
   let either = x or y
   let negated = not x

✅ Control Flow
   if x > 5:
       print("big")
   else:
       print("small")
   
   while i < 10:
       i = i + 1

⏳ Functions (partial)
   fn add(a, b):
       return a + b
   # Works in interpreter, not in compiler yet

❌ Not Yet Implemented
   - For loops (parser support needed)
   - Arrays/lists
   - Strings (partial)
   - Classes/structs
   - Imports/modules
   - Async/await
   - Generics (type system ready, codegen pending)
