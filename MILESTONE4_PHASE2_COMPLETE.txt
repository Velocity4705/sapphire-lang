TECHNICAL HIGHLIGHTS
================================================================================

1. Mark-and-Sweep Algorithm
```cpp
void collect() {
    // Mark phase
    mark_roots();
    process_work_queue();
    
    // Sweep phase
    sweep();
}
```

2. Root Scanning
```cpp
void scan_stack() {
    void* sp = get_stack_pointer();
    scan_memory(sp, stack_bottom_ - sp);
}
```

3. Conservative Marking
```cpp
// Treat every word as potential pointer
for (void** ptr = start; ptr < end; ptr++) {
    if (is_heap_pointer(*ptr)) {
        mark_object(*ptr);
    }
}
```

4. Adaptive Threshold
```cpp
// Double threshold after each GC
gc_threshold_ = bytes_at_last_gc_ * 2;
```

================================================================================
CURRENT LIMITATIONS
================================================================================

Known Limitations (by design for Phase 2):

1. Conservative Scanning
   - Scans all memory as potential pointers
   - May keep some garbage alive
   - Will be improved with type information

2. Stop-the-World
   - Pauses program during GC
   - Very fast (119μs) so not a problem
   - Can be made concurrent later

3. No Generational GC
   - Scans all objects every time
   - Still very fast
   - Can be added later

4. No Compaction
   - Doesn't move objects
   - May have fragmentation
   - Can be added later

None of these are problems for current use! ✅
