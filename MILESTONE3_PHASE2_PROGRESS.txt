================================================================================
LESSONS LEARNED
================================================================================

1. AST Structure Matters
   - Operators stored as strings, not tokens
   - LiteralExpr has its own Type enum
   - Need to adapt to existing AST design

2. LLVM 21 Changes
   - Opaque pointers (no getPointerElementType)
   - Need to track types separately
   - PointerType::get(*context, 0) for generic pointers

3. Type Tracking Essential
   - Must track variable types for loading
   - Symbol table needs both value and type
   - Makes code generation cleaner

4. Incremental Development Works
   - Start with simple expressions
   - Add complexity gradually
   - Test each component
